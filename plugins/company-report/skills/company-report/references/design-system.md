# Design System

첨부된 레퍼런스의 시각 문법을 공개 가능한 범용 템플릿으로 정리한 기준이다. 원본 스크린샷, 로고, 주소, 회사명은 포함하지 않는다.

## 우선순위

1. 사용자가 제공한 실제 회사 PPTX와 브랜드 가이드
2. 사용자가 승인한 별도 디자인 지시
3. 번들 `assets/template.pptx`와 이 문서의 기본값

상위 기준과 충돌하는 하위 기준은 적용하지 않는다.

## 캔버스와 그리드

- 화면 비율: 16:9
- 기본 캔버스: 1280 × 720px
- 좌우 안전 여백: 64~72px
- 상단 제목 기준선: 48~56px
- 본문 시작선: 128~152px
- 하단 안전 여백: 44~52px
- 열 간격: 24~32px
- 1단, 2단, 3단 중 내용량에 맞는 가장 단순한 구성을 고른다.
- 열 구획은 얇은 회색 선 또는 충분한 공백으로 표현한다.

## 브랜드 비종속 시각 요소

- 배경: `#FFFFFF`
- 기본 텍스트: `#171717`
- 보조 텍스트: `#5F6368`
- 구분선: `#D8DADD`
- 강조 빨강: `#F04444`
- 보조 파랑: `#168BD2`
- 하단 포인트 보라: `#7B61FF`
- 슬라이드 왼쪽 상단에 6~8px 너비의 짧은 빨강 세로선을 둔다.
- 하단에는 5~7px 높이의 보라 포인트 선을 선택적으로 둔다.
- 로고가 없으면 임의 로고를 만들지 말고 텍스트형 푸터만 사용한다.

## 타이포그래피

- 선호 서체: 나눔바른고딕
- 대체 순서: Noto Sans KR, Apple SD Gothic Neo, Aptos, sans-serif
- 표지 제목: Bold 34pt 내외
- 본문 슬라이드 제목: Bold 30pt 내외
- 부제: Regular 18~20pt
- 섹션 헤더: Bold 17~18pt
- 본문: Regular 12~15pt
- 각주와 페이지 표기: Regular 9~10pt
- 한 줄 제목이 두 줄로 감기면 글자를 줄이기 전에 문구를 축약한다.
- 본문은 12pt 아래로 축소하지 않고 필요하면 백데이터로 분리한다.

## 제목과 메시지

- 제목에는 한 문장의 결론 또는 판단을 쓴다.
- 한 슬라이드에서 시각적으로 가장 강한 문구는 하나만 둔다.
- 섹션 헤더는 명사형 주제보다 “무엇이 왜 중요한가”를 드러내게 쓴다.
- 핵심 수치는 단위, 기간, 비교 기준과 함께 크게 표시한다.
- 강조색은 결론, 변화량, 요청 사항처럼 실제 의미가 있는 항목에만 사용한다.

## 레이아웃 패턴

### 1페이지 요약

- 상단: 결론형 제목과 한 줄 보조 메시지
- 중단: 배경/현황, 핵심 근거, 요청/다음 단계의 3개 논리 구역
- 하단: 출처, 기준일, 작성 부서, 페이지 번호
- 세 구역은 카드처럼 과도하게 꾸미지 말고 공백과 얇은 구분선으로 나눈다.

### 2단

- 비교, 현황/계획, 문제/대응, 선택지 A/B에 사용한다.
- 열 너비는 기본 1:1이며 한쪽 메시지가 더 중요하면 최대 2:1까지 허용한다.

### 3단

- 세 가지 근거, 실행축, 옵션 비교에 사용한다.
- 각 열의 제목 높이와 첫 본문 기준선을 맞춘다.
- 세 열에 동일한 정보 위계를 유지한다.

### 백데이터

- 표와 차트는 결론을 검증하는 데 필요한 항목만 남긴다.
- 표 머리글은 짙은 회색 또는 아주 연한 회색 배경을 사용한다.
- 출처와 산식은 9~10pt 각주로 표시한다.

## 검수 기준

- 모든 객체가 안전 여백 안에 있는지 확인한다.
- 제목과 본문이 의도치 않게 감기지 않았는지 확인한다.
- 열 간격, 기준선, 푸터 위치가 슬라이드 간 일관적인지 확인한다.
- 강조 빨강과 파랑을 장식 목적으로 남용하지 않는다.
- 실제 회사 정보나 원본 스크린샷이 번들 자산에 들어가지 않았는지 확인한다.
