# Report Types

보고 목적에 맞는 1페이지 구조를 선택하고, 필요한 경우에만 백데이터를 붙인다.

## 공통 원칙

- 첫 장만 읽어도 결론, 근거, 요청 사항을 이해할 수 있게 한다.
- 1페이지 요약은 항상 1장으로 유지한다.
- 세부 데이터가 요약의 메시지를 흐릴 때만 뒤로 분리한다.
- 보고 대상의 의사결정 권한과 예상 질문을 기준으로 정보량을 조절한다.

## 신규 기획·제안

**의사소통 목표:** 왜 지금 추진해야 하며 무엇을 승인해야 하는지 납득시킨다.

1페이지 기본 구조:

1. 기회 또는 문제: 현재 상황과 시급성
2. 제안: 해결 방식과 차별점
3. 기대 효과: 정량·정성 가치
4. 실행안: 일정, 담당, 주요 리스크
5. 요청: 승인 범위, 예산, 다음 단계

백데이터 후보: 시장 근거, 상세 일정, 비용 산식, 대안 비교, 리스크 대응표.

## 정기 진행 보고

**의사소통 목표:** 계획 대비 상태를 빠르게 이해하고 필요한 지원을 결정하게 한다.

1페이지 기본 구조:

1. 종합 상태: 정상/주의/위험과 한 줄 판단
2. 핵심 지표: 목표, 실적, 차이, 추세
3. 완료 사항: 이번 기간의 의미 있는 진척
4. 이슈와 대응: 영향, 조치, 담당, 기한
5. 다음 기간: 우선순위와 필요한 지원

백데이터 후보: 세부 작업 목록, 지표 추이, 일정표, 이슈 로그.

## 문제·이슈 보고

**의사소통 목표:** 영향과 원인을 분명히 하고 즉시 필요한 결정을 끌어낸다.

1페이지 기본 구조:

1. 이슈 요약: 무엇이 언제 발생했는가
2. 영향: 고객, 매출, 일정, 운영에 미치는 범위
3. 원인: 확인 사실과 가설을 구분
4. 현재 대응: 완료, 진행, 미착수 조치
5. 요청: 의사결정, 자원, 커뮤니케이션 필요사항

백데이터 후보: 타임라인, 원인 분석, 영향 산식, 상세 조치 목록.

## 결과·성과 보고

**의사소통 목표:** 목표 대비 성과와 학습을 설명하고 후속 방향을 합의한다.

1페이지 기본 구조:

1. 핵심 결론: 목표 달성 여부와 의미
2. 성과: 핵심 지표와 기준 대비 변화
3. 기여 요인: 성과를 만든 주요 활동
4. 한계와 학습: 미달 영역과 원인
5. 후속 제안: 유지, 확대, 중단, 보완 계획

백데이터 후보: 지표 정의, 세그먼트 분석, 실험 결과, 비용 대비 효과.

## 의사결정 요청

**의사소통 목표:** 선택지와 판단 기준을 명확히 해 결정을 받는다.

1페이지 기본 구조:

1. 결정 질문: 오늘 무엇을 정해야 하는가
2. 배경과 제약: 결정이 필요한 이유와 경계 조건
3. 선택지: 가능한 대안과 핵심 차이
4. 권고안: 추천 선택지와 근거
5. 요청: 결정 기한, 승인자, 결정 후 행동

백데이터 후보: 평가 기준표, 민감도 분석, 비용 산식, 리스크 비교.

## 백데이터 판단 기준

다음 중 하나라도 해당하면 백데이터를 제안한다.

- 핵심 수치의 산식이나 원자료 확인이 필요하다.
- 선택지가 셋 이상이고 비교 기준이 많다.
- 일정, 조직, 리스크, 비용을 첫 장에 모두 넣으면 결론이 묻힌다.
- 예상 질문에 답하려면 상세 표나 추이가 필요하다.

장수는 필요한 질문 수에 맞춰 최소화하고, 각 장의 목적을 구조 승인 단계에서 밝힌다.
